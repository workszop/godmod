Introduction

The GodMod Individual Beings Extension is an add-on to the GodMod Library that allows for the creation and manipulation of individual entities within your simulated universes. This extension provides granular control over the lives of beings, including their luck, fate, skills, relationships, and life events.

With this extension, you can: - Create and manage individual beings of any species - Control the luck and fate of individuals - Manage skills and relationships - Simulate the passage of time and generate life events - Integrate individual-level control with universe-level manipulation

Installation

To use the GodMod Individual Beings Extension, ensure you have the main GodMod Library installed. Then, include the extension file in your project directory.

from godmod import *
from godmod_individual_beings import Individual, IndividualManager, LifeSimulator

Classes

Individual

The Individual class represents a single being in the universe.

Attributes:

  • name (str): The name of the individual
  • species (str): The species of the individual
  • birthdate (datetime): The birth date of the individual
  • luck (int): A value from 0-100 representing the individual's luck
  • fate (dict): Key events predestined in the individual's life
  • skills (dict): The individual's skills and their levels
  • relationships (dict): The individual's relationships with other individuals
  • life_events (list): A list of significant events in the individual's life
  • is_alive (bool): Indicates whether the individual is alive

Methods:

  • age(): Returns the current age of the individual
  • set_luck(value: int): Sets the luck value for the individual
  • add_fate_event(event: str, age: int): Adds a predestined event to the individual's fate
  • add_skill(skill: str, level: int): Adds a new skill to the individual
  • improve_skill(skill: str, improvement: int): Improves an existing skill
  • add_relationship(other: Individual, relationship_type: str): Adds a relationship with another individual
  • add_life_event(event: str): Adds a life event to the individual's history
  • die(): Marks the individual as deceased

IndividualManager

The IndividualManager class manages the creation and tracking of individuals in the universe.

Methods:

  • create_individual(name: str, species: str, birthdate: datetime) -> Individual: Creates and returns a new individual
  • get_individual(name: str) -> Individual: Retrieves an individual by name
  • remove_individual(name: str): Removes an individual from the universe

LifeSimulator

The LifeSimulator class provides methods to simulate the passage of time and generate life events for individuals.

Methods:

  • simulate_day(individual: Individual): Simulates a day in the life of an individual, potentially triggering events based on luck, fate, and random chance

Usage Examples

Creating and Managing Individuals

manager = IndividualManager()

# Create individuals
alice = manager.create_individual("Alice", "Human", datetime(2000, 1, 1))
bob = manager.create_individual("Bob", "Human", datetime(1995, 5, 15))

# Set attributes
alice.set_luck(75)
bob.add_fate_event("Becomes a famous chef", 35)

# Add skills
alice.add_skill("Programming", 5)
bob.add_skill("Cooking", 8)

# Create relationships
alice.add_relationship(bob, "Friend")
bob.add_relationship(alice, "Friend")

# Retrieve an individual
retrieved_alice = manager.get_individual("Alice")

# Remove an individual
manager.remove_individual("Bob")

Simulating Life Events

simulator = LifeSimulator()

# Simulate a day in Alice's life
simulator.simulate_day(alice)

# Simulate multiple days
for _ in range(365):
    simulator.simulate_day(alice)

# Check Alice's life events
print(alice.life_events)

Integration with GodMod

The Individual Beings Extension can be integrated with other GodMod modules to create more complex and interesting scenarios:

from godmod import Universe, CelestialBody, Weather
from godmod_individual_beings import Individual, IndividualManager, LifeSimulator

# Create a universe and a planet
universe = Universe("MyUniverse")
earth = CelestialBody("Earth", "planet")
universe.add_body(earth)

# Create weather on the planet
weather = Weather(earth)

# Create individuals on the planet
manager = IndividualManager()
alice = manager.create_individual("Alice", "Human", datetime(2000, 1, 1))

# Simulate a day with weather events
weather.set_rain(duration=60)
LifeSimulator.simulate_day(alice)

# Alice's life events might now include weather-related incidents
print(alice.life_events)

Best Practices

  1. Memory Management: When dealing with large numbers of individuals, be mindful of memory usage. Remove individuals from the IndividualManager when they are no longer needed.

  2. Balancing Luck: The luck attribute significantly affects an individual's life events. Balance it carefully to maintain realism and interest in your simulation.

  3. Expanding Events: Consider expanding the LifeSimulator.simulate_day() method to include more varied and complex events for a richer simulation.

  4. Integration: Integrate individual-level events with universe-level events for a more cohesive and interesting simulation.

  5. Customization: Extend the Individual class to add more attributes or methods specific to your simulation needs.

Troubleshooting

  • High Memory Usage: If you're experiencing high memory usage, ensure you're removing unnecessary individuals and not creating too many instances.
  • Unrealistic Simulations: If simulations seem unrealistic, review the probabilities in the LifeSimulator.simulate_day() method and adjust as needed.
  • Errors in Relationship Management: Ensure that when you remove an individual, you also update the relationships of other individuals who were connected to them.
